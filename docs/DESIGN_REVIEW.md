# Design Review

GraduationLion의 UI는 명확한 학교 서비스 톤과 단순한 정보 구조를 가진 MVP 디자인입니다. 포트폴리오에서는 "완성도 높은 상용 UI"보다는 "졸업 요건 확인이라는 목적에 맞춰 필요한 정보를 빠르게 보여주는 앱"으로 설명하는 편이 자연스럽습니다.

## Strengths

- 메인 컬러가 일관됩니다. 한동대학교 이미지를 떠올리게 하는 진한 블루를 로그인, 선택 상태, 진행률, 주요 버튼에 반복적으로 사용합니다.
- 주요 화면 흐름이 단순합니다. 로그인, 초기 정보 입력, 홈, 추천, 마이페이지로 이어지는 구조가 이해하기 쉽습니다.
- 정보 구분이 분명합니다. 학기별 구분, 수강 완료/미완료, 전공/교양 탭처럼 사용자가 찾는 기준이 화면에 직접 드러납니다.
- 모바일 앱으로서 최소한의 사용성은 갖췄습니다. 하단 내비게이션, 검색창, 리스트, 다이얼로그 등 익숙한 패턴을 사용합니다.

## Limitations

- 화면별 간격과 텍스트 스타일이 완전히 체계화되어 있지는 않습니다. 같은 성격의 리스트/섹션이라도 padding, font weight, height가 조금씩 다릅니다.
- `Container` 중심의 UI가 많아 컴포넌트 재사용성이 낮습니다. 학기 헤더, 과목 리스트 아이템, 상태 요약 카드 등을 별도 위젯으로 분리하면 코드와 UI가 같이 깔끔해집니다.
- 검색창의 배경과 텍스트가 모두 밝은 톤이라 대비가 약합니다. 실제 사용성 기준에서는 입력 텍스트 색상과 hint 색상을 더 명확히 나누는 편이 좋습니다.
- 앱 전반이 흰색/회색/블루 중심이라 안정적이지만, 추천 화면이나 졸업 진행률 화면에서 정보 우선순위를 더 시각적으로 강조할 여지가 있습니다.
- 영어/한국어 문구가 섞여 있습니다. 사용자-facing 문구는 한국어 중심으로 통일하면 더 완성도 있어 보입니다.

## Practical Improvement Ideas

- `AppColors`처럼 `AppTextStyles`, `AppSpacing`을 추가해 디자인 토큰을 조금 더 체계화합니다.
- 과목 리스트 아이템, 학기 섹션 헤더, 진행률 영역을 재사용 위젯으로 분리합니다.
- 로그인 실패, 중복 과목 추가 실패 같은 snackbar 문구를 한국어로 통일합니다.
- 추천 화면에서 필수/선택 추천 과목의 시각적 구분을 강화합니다.
- 홈 화면의 졸업 요건 진행률은 숫자뿐 아니라 상태 라벨을 함께 제공하면 이해가 쉬워집니다.

## Current Design Level

현재 디자인은 기능 검증을 위한 MVP UI로는 충분하지만, 상용 앱 수준의 디자인 시스템을 갖춘 상태는 아닙니다. 다만 핵심 색상과 화면 흐름이 이미 잡혀 있기 때문에, 디자인 토큰과 재사용 위젯을 조금만 추가해도 포트폴리오에서 "기능 중심 앱을 정리하고 UI 일관성을 개선했다"라고 설명하기 좋은 상태입니다.
